title: Commit And Push
description: Review changes, create a conventional commit, and push to the current branch.
---

Please help me commit and push my changes to the current branch.

## Steps:

1. **Review the changes**: Run `git status` and `git diff` to understand what has been modified.

2. **Analyze the changes**: Determine the appropriate conventional commit type based on the changes:
   - `feat`: A new feature
   - `fix`: A bug fix
   - `docs`: Documentation only changes
   - `style`: Code style changes (formatting, etc.)
   - `refactor`: Code changes that neither fix bugs nor add features
   - `perf`: Performance improvements
   - `test`: Adding or correcting tests
   - `chore`: Build process or tooling changes

3. **Create the commit**: Stage all relevant changes and create a commit with a conventional commit message in the format `<type>(<scope>): <subject>`

4. **Push to current branch**: Push the commit to the remote repository on the current branch.

Please proceed with reviewing the changes and creating an appropriate commit message.
